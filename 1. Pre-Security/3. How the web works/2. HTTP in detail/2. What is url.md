A URL is predominantly an instruction on how to access a resource on the internet. The below image shows what a URL looks like with all of its features (it does not use all features in every request).

scheme: tells the protocol to be used on accessing web content
user: some services require auth to log in(you can put them in the url)
host: domain name of the ip address
port: port where the service resides
path: the part of the web application you wish to access

the rest are query strings and fragments etc.
